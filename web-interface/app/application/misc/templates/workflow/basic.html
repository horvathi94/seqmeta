Basic

This application is designed to manage sequencing metadata and format it for upload to GISAID, ENA and NCBI databases.

To start the workflow, first please register:

  1. Authors, people who worked on sequencing the sample and people who collected the samples.
  2. Author groups, which are formed by the authors, who contributed to the sequencing. These groups may only contain authors registered in the first step.
  3. Institutions, the names and addresses of the institutions or organizations, which collected, sequenced or submitted the samples.
  4. Add needed extra values, the fields needed for metadata upload are initialised with general values or values found in examples of the, but you may need to add some custom values, specific to your workflow.
  5. Edit the default values, For an easier and faster editing experience it is recommanded that you edit these values to fit your workflow.
  6. Edit virusname and isolate name, if you haven't already edited the virsuname (required GISAID) and the isolate name (required by ENA and NCBI) please do so. These values are meant to speed up your editing, by programatically generating the virus and isolate names for the samples.

    7. You are now ready to add your first samples!

    Note: When adding or editing samples be carefull to complete all fields that are required for the desired submission. The requirement levels are represented with superscript markers.

    After you are finished with adding the samples, select the samples wich you wish to upload and select the button, which represents the upload server and download the zip file, which contains all the files needed to submit the samples. The upload workflow of these files are described in more detail on these pages:

    • GISAID
    • ENA
    • NCBI

    Note: Before uploading the raw read data clean any human identifiable reads. For this you can use this program.
